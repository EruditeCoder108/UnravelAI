Causal Chain Detection

See how bugs
actually spread.

The AST engine connects async triggers to downstream state writes — deterministically, without LLM inference. It knows when a fetch result could overwrite stale state.

FN
useEffect(() => { ... }, [userId]) Effect fires when userId changes
│
ASYNC
fetch('/api/user/' + userId) Async boundary — promise enters queue [L14]
│   — userId changes again here —
WRITE
setUserData(response.json()) State write after async [L22]
│
⚠
No cancellation guard detected Response from old userId overwrites new userId data
✓
Fixed: AbortController pattern controller.abort() in cleanup — guard confidence: HIGH

How Unravel detects this

The AST engine tracks every timing node (fetch, then, catch) and every state write (set[A-Z]* pattern, dispatch). When a write follows a timing trigger in the same function, it builds a causal pair.

Three-strategy guard detection

Strategy 1 (HIGH): AbortController — detects `new AbortController()` variable read between trigger and write. Strategy 2 (HIGH): Boolean flag — `let cancelled = false` written in same function, read as condition. Strategy 3 (LOW confidence): Substring heuristic on variable names as fallback.

What the LLM receives

The AI sees verified causal structure, not raw code. "fetch() [L14] → setUserData() [L22] — no cancellation guard" is a deterministic fact that the model can reason about without guessing.

OUTPUT IN AST CONTEXT
Causal Chains (async → state write):
  fetch() [L14] [useData.jsx] → setUserData() in fetchUser [L22]
  ⚠ no cancellation guard — stale write risk
  ⚠ async inside useEffect with no cleanup return
Semantic Mutation Tracking

Not just where.
What kind.

Unravel distinguishes between a variable being replaced entirely versus a property being mutated on a shared object. The difference determines whether the bug affects other references.

Before — Flat "written" for everything
// Code under analysis:
let session = {'{'}id: 1, active: true{'}'}
session = getNewSession()   // replaces entirely
session.active = false     // mutates property
session.id += 1           // augmented assign

// Old AST output (ambiguous):
session
  written: fn L12, fn L13, fn L14 ← all same

// Verifier confused: same object or new one?
⚠ false dependency warning generated
After — Semantic types, full picture
// Same code, new AST output:

session
  reassigned: fn L12     ← new object entirely

session.active
  written: fn L13 ← property write

session.id
  written: fn L14 ← property write
    (augmented assignment: +=)

✓ Verifier knows: L12 breaks old references
✓ L13, L14 mutate the current object
Features

Built for
real codebases.

Every feature exists because a real bug required it — not because it looked impressive on a product page.

🌳

Tree-sitter WASM Engine

Replaced Babel entirely. Handles syntax errors gracefully — returns a partial tree instead of failing. Supports JS, JSX, TS, TSX with the same parser. Custom scope resolver built from scratch because Babel's API doesn't exist in tree-sitter.

AST Layer
🔗

Cross-File AST Resolution

Traces imports and exports across files. Builds a module map, resolves symbol origins, merges mutation chains across file boundaries, and emits risk signals for cross-file state mutations.

Analysis
🚦

Graph-Frontier Router

BFS traversal over import/call/mutation graphs. From 8,252 VS Code files to 4 relevant ones. Zero overhead on small projects. Every result includes `_provenance.routerStrategy` — self-reporting what happened.

Routing
⛔

Solvability Check

The feature no other debugger has. Before generating a fix, checks if the bug is actually fixable from inside your code. Layer Boundary verdict for OS/browser/API upstream bugs. No more wrong patches for impossible problems.

New in v3.3
⛓

Causal Chain Detection

Deterministic detection of async→state write patterns. Three-strategy guard detection: AbortController, boolean flag, substring fallback. Each chain includes guard confidence. No LLM inference required.

New in v3.3
🔬

Claim Verifier

Every LLM claim is cross-checked against AST data and provided files. File references verified. Line numbers range-checked. Variable names matched. Fabricated evidence is caught. Confidence is penalized automatically.

Trust Layer
⚡

Progressive Streaming

All three providers (Anthropic, Gemini, OpenAI) stream results via SSE. Sections appear as they generate. Chunk-count parse trigger. `lastHash` dedup prevents duplicate emissions. Zero regression risk — activates only when callback provided.

UX
🧬

Semantic Mutation Types

Every write is classified: `reassigned` (variable gets new value) vs `written` (property mutated on existing object). Covers augmented assignments, destructuring, update expressions, computed property mutations.

New in v3.3
🛡

Anti-Sycophancy Engine

7 hardcoded rules the model cannot override. If code is correct, it says so. The crash site is never the root cause. Variable names don't prove behavior. 3 mutually exclusive hypotheses — eliminate, don't converge.

Analysis Modes

One engine.
Three lenses.

Each mode uses a different system prompt and output schema. The AST pre-analysis runs for all three — mutations are as relevant to security as they are to bugs.

🐛

Debug Mode

The full 9-phase pipeline. Three competing hypotheses, elimination by evidence, exact file and line, minimal surgical fix.

  • Root cause with evidence
  • Variable state table
  • Execution timeline
  • Hypothesis tree (eliminated vs survived)
  • Concept extraction — "what did this bug teach?"
  • AI fix prompt for Cursor/Bolt

Architecture walkthrough for codebases you didn't write. Entry points, data flow, layer structure, dependency map.
