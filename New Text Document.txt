

# 🧠 “Concept Extraction” (Learning From the Bug)

Instead of only fixing the bug, Unravel would automatically extract the **underlying programming concept** that caused the failure and teach it to the user.

So the system answers three levels:

```text
1. What broke
2. Why it broke
3. What programming concept this bug belongs to
```

That third level is where almost every tool stops today.

---

# How It Would Work

After diagnosing the bug, Unravel would map it to a **concept category**.

For example, the timer bug you tested belongs to:

```text
Concept: State Mutation + Temporal Logic
```

Unravel would say something like:

> This bug occurred because the program mutated the variable `duration`, which should represent the constant total session time. When `pause()` changed it to `remaining`, the timer’s invariant was violated. This is a classic **state mutation bug**.

Then it would show the user a short conceptual lesson.

---

# Example From Your Timer Bug

### Bug Identified

```javascript
duration = remaining
```

### Concept Extracted

```text
State Mutation Bug
```

### Concept Explanation

```
In many programs, some variables represent fixed configuration values.
If these values are mutated during execution, calculations that depend on them become incorrect.

In this timer:
duration = total session time
remaining = time left

Mutating duration breaks the equation:
remaining = duration - elapsed
```

Sam, the feature that would make **Unravel feel truly magical**—and that almost no AI debugger currently has—is something I’d call:

# 🔎 “Bug Replay” (Causal Debugging Playback)

Instead of just explaining the bug, Unravel would **replay the execution of the program step-by-step**, showing exactly **how the bug happened**.

Think of it like **a crime scene reconstruction for code**.

---

# What the User Experiences

Instead of only seeing a text explanation, the user would see something like:

```
▶ Execution Replay
t0  start() called
t1  interval created
t2  tick() executed
t3  pause() called
t4  duration mutated
t5  start() called again
t6  elapsed calculation wrong
t7  timer jumps forward
```

And each step would show **variable values changing**.

Example:

```
duration = 1500
remaining = 1487
startTimestamp = 1710000123
```

Then the next step:

```
pause()

duration = 1487  ← BUG
remaining = 1487
```

The system literally **highlights the exact moment the program breaks**.

---

# Why This Feels Magical

Because debugging normally requires mentally doing this:

```
read code
simulate program in your head
track variable states
```

That’s extremely hard for beginners.

Bug Replay does the simulation **visually**.

So the user immediately sees:

> “Oh… this variable changed here, and that broke everything.”

# Example for Your Timer Bug

Unravel could show something like:

```
SESSION START
duration = 1500
remaining = 1500
startTimestamp = 1000
```

```
tick()
elapsed = 5
remaining = 1495

```

```
remaining = 1495
```

Then:


And the bug becomes **obvious instantly**.
